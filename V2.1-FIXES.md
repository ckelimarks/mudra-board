# V2.1 - Bug Fixes & Gesture Commands

## Fixed Issues

### 1. Text Dragging Now Works ✅

**Problem:** Couldn't grab and move text after placing it

**Root cause:** Hit detection was using `document.elementsFromPoint()` which expects viewport coordinates, but SVG overlay had `pointer-events: none` making hit detection fail.

**Solution:** Rewrote hit detection to directly check SVG text element bounding boxes:
```javascript
getTextElementAt(x, y) {
  const textElements = this.getAllTextElements();
  for (const text of textElements) {
    const bbox = text.getBBox();
    // Check if point is inside bbox with margin for easier grabbing
    if (x >= bbox.x - 10 && x <= bbox.x + bbox.width + 10 &&
        y >= bbox.y - 10 && y <= bbox.y + bbox.height + 10) {
      return text;
    }
  }
  return null;
}
```

**Test:**
1. Place some text (speech on, say something, pinch to place)
2. Make pinch gesture over the text
3. Move hand - text should follow
4. Release pinch - text drops at new position

### 2. Gesture-Based Speech Toggle ✅

**Problem:** Had to click button to turn speech on/off - not hands-free

**Solution:** Added second-hand gesture detection:
- Show **open palm** with your **other hand** → toggles speech on/off
- Green indicator appears at wrist when open palm detected
- 🎤 emoji shows when gesture triggers
- 1 second cooldown prevents rapid toggling

**How it works:**
1. System now tracks up to 2 hands (was 1)
2. Primary hand (first detected) = drawing/dragging hand
3. Secondary hand = gesture command hand
4. Open palm detection: all 5 fingers extended + spread apart
5. On transition from "not open palm" → "open palm": toggle speech

**Implementation:**
```javascript
// Detect open palm
detectOpenPalm(landmarks) {
  const allFingersExtended =
    index.tip.y < index.pip.y &&
    middle.tip.y < middle.pip.y &&
    ring.tip.y < ring.pip.y &&
    pinky.tip.y < pinky.pip.y;

  const fingersSpread = Math.abs(index.x - pinky.x) > 0.15;

  return allFingersExtended && fingersSpread;
}
```

**Test:**
1. Have one hand in view (primary hand)
2. Bring your other hand into view
3. Open palm fully (fingers spread)
4. Speech should toggle on/off
5. Green circle with 🎤 appears at wrist when detected
6. Can toggle on/off repeatedly by hiding hand and showing palm again

## Known Limitations

1. **Open palm detection** requires good hand visibility and lighting
2. **Two hands** tracked but only primary hand can draw/drag (secondary is for commands only)
3. **Gesture cooldown** means you can't toggle speech faster than once per second
4. **Text grabbing** requires pinching directly over text bbox (hit area has 10px margin)
